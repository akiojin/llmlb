# クイックスタート: Playground Chat マルチモーダル対応（SPEC-5fc9fe92）

## 目的

Playground Chat でテキスト・画像・音声を使った入力ができることを、人手で短時間に確認する。

## 事前準備

- LLM Load Balancer を起動し、Playground にアクセスできること
- モデルを選択できること（画像/音声対応モデルはUI上で表示される）

## 画像（P1）の確認手順

1. `/playground` を開く
2. モデルを選択する
3. 画像添付ボタンから PNG/JPEG を選択する
4. プレビューが表示されることを確認する
5. 送信する
6. 会話ログに画像とユーザー入力が表示され、回答が返ることを確認する

### 画像の貼り付け確認

1. クリップボードに画像をコピーする（例: スクリーンショットをコピー）
2. Playground の入力エリアにフォーカスし貼り付け（ペースト）する
3. 画像が添付されプレビューが表示されることを確認する

### 画像のエラー確認

- 画像非対応モデルを選択した場合、添付ボタンが無効化される（または理由が表示される）
- サイズ上限超過/形式不正の場合、送信されず分かりやすいエラーが表示される

## 音声（P2）の確認手順

1. `/playground` を開く
2. モデルを選択する
3. 音声添付ボタンから WAV/MP3 等を選択する
4. 音声プレイヤーが表示されることを確認する
5. 送信する
6. 会話ログに音声とユーザー入力が表示され、回答が返ることを確認する

### 音声の貼り付け確認

1. クリップボードに音声ファイルをコピーする
2. Playground の入力エリアにフォーカスし貼り付け（ペースト）する
3. 音声が添付され、音声プレイヤーが表示されることを確認する

### 音声のエラー確認

- 音声非対応モデルを選択した場合、添付ボタンが無効化される（または理由が表示される）
- サイズ上限超過/形式不正の場合、送信されず分かりやすいエラーが表示される

## 生成結果メディア（P2）の確認手順

1. アシスタントの回答本文に画像URL/音声URL（またはデータURL）が含まれる状態を作る
2. 会話ログ上で、画像がプレビュー表示されること、音声がプレイヤー表示されることを確認する
3. 読み込みに失敗する場合でも、リンクを開く等の次の行動が取れることを確認する

## テキスト互換（P3）の確認手順

1. 添付なしでテキストのみを送信する
2. 送信、ストリーミング、停止が従来どおり利用できることを確認する
