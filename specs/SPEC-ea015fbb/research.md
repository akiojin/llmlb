# リサーチ: Web UI 画面一覧

## 調査目的

LLM Load Balancer Web UIの全画面構成と画面遷移を把握し、索引ドキュメントとして整理する。

## 画面構成の調査

### 認証系画面

| 画面 | パス | 役割 |
|------|------|------|
| ログイン | `/dashboard/login.html` | JWT認証によるログイン |
| ユーザー登録 | `/dashboard/register.html` | 招待コードによる新規登録 |

### メイン画面

| 画面 | パス | 役割 |
|------|------|------|
| ダッシュボード | `/dashboard` | システム全体の監視・管理 |
| Playground | `/playground` | マルチモーダル対応チャットUI |

## ダッシュボード内セクション構成

### セクション一覧

1. **Header (SEC-001)**: ナビゲーション、テーマ切替、ログアウト
2. **StatsCards (SEC-002)**: ノード数、リクエスト数などの統計
3. **NodeTable (SEC-003)**: ノード一覧・状態・GPU情報
4. **RequestHistoryTable (SEC-004)**: ページネーション付きリクエスト履歴
5. **LogViewer (SEC-005)**: ロードバランサーログ表示
6. **ModelsSection (SEC-006)**: モデル一覧・割り当て・登録
7. **CloudProvidersSection (SEC-007)**: クラウドプロバイダー状況

## モーダル構成

| モーダル | 親画面 | 用途 |
|----------|--------|------|
| NodeSettingsModal | ダッシュボード | ノード設定編集 |
| NodeDeleteConfirm | ダッシュボード | ノード削除確認 |
| RequestDetailModal | ダッシュボード | リクエスト詳細表示 |
| ModelRegisterModal | ダッシュボード | モデル登録フォーム |

## 画面遷移パターン

### 認証フロー

```text
未認証 → ログイン → ダッシュボード
            ↓
       ユーザー登録 → ログイン
```

### メイン画面間遷移

```text
ダッシュボード ↔ Playground (1クリック)
ダッシュボード → ログアウト → ログイン
```

## 設計原則

### ナビゲーション要件

- 任意の画面から主要画面へ2クリック以内で遷移可能
- 共通Headerコンポーネントで一貫したナビゲーション
- ブラウザの戻る/進むボタン対応

### レスポンシブ対応

- モバイル / タブレット / デスクトップ
- 画面サイズに応じたレイアウト調整
